# What is SAP Event Mesh?

## Overview

**SAP Event Mesh** allows applications to communicate through **asynchronous events**.  
It enables greater agility and scalability by allowing you to build **responsive applications** that operate independently while participating in **event-driven business processes** across your business ecosystem.

[More Information](https://help.sap.com/docs/event-mesh/event-mesh/what-is-sap-event-mesh?locale=en-US)

---

## Features

### 🟢 Publish Business Events
- Publish events from **SAP** and **non-SAP** sources across hybrid landscapes.  
- Connect your **digital core** (such as SAP Cloud ERP) to **extension applications** through an **event-driven architecture**.  
- Facilitate real-time, decoupled communication for improved responsiveness.

### 🟣 Consume Business Events
- Consume events from **SAP** and **non-SAP** sources across the **SAP event-driven ecosystem**.  
- Integrate seamlessly with:
  - **SAP Extension Suite**  
  - **SAP Integration Suite**  
  - Selected inbound-enabled SAP backends  
- React to business events automatically to streamline processes and reduce latency.

### 🔵 Connect Seamlessly
- Achieve **reliable data transmission** for extension and integration scenarios.  
- Enable **decoupled communication** between systems for flexibility and scalability.  
- Support **hybrid integration** without dependency on synchronous connections.

---

## Environment

SAP Event Mesh is available in the **Cloud Foundry environment** of **SAP Business Technology Platform (SAP BTP)**.  
It provides the foundation for **event-driven integration** between SAP and non-SAP systems, ensuring scalability and reliability across your enterprise landscape.

---

## Key Benefits

- Build **event-driven applications** that respond to changes in real time.  
- Improve **system scalability** and **loose coupling**.  
- Extend and integrate SAP solutions using **modern messaging patterns**.  
- Support **hybrid and cloud-native architectures** on SAP BTP.
